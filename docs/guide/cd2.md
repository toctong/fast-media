# CloudDrive2 配置

入口：**插件中心 → CloudDrive2 助手**

CD2 负责把网盘挂载成路径，供链接同步、归档搬运、深度删除等调用。Docker 部署时请把宿主机的 CD2 挂载目录映射进容器（见 [部署](./deploy.md)）。

## 必填字段

| 字段 | 说明 |
|------|------|
| 启用 | 开启健康检测；失败时归档 / 链接同步等可能暂停 |
| 服务地址 | CD2 Web，常见 `http://192.168.x.x:19798` |
| 登录用户名 / 密码 | CD2 账号 |
| 挂载根路径 | 如 `/CloudNAS/CloudDrive`；可「自动获取」 |

点 **测试连接** → 保存。

> 在 Docker 里不要填 `http://127.0.0.1:19798` 除非 CD2 与 Fast Media 在同一网络命名空间且指向正确；优先填**宿主机局域网 IP**。

## 和卷映射的关系

若 compose 中有：

```yaml
- /vol1/1000/cd2/CloudNAS:/CloudNAS
```

则「挂载根路径」通常对应容器内能看到的前缀，例如 `/CloudNAS/...`。以你 CD2 实际挂载名为准。

## 下游会用到 CD2 的功能

| 功能 | 说明 |
|------|------|
| 链接同步 STRM | 写云盘路径 / HTTP 链 |
| 归档任务 | 处理方式可选 cd2 移动 / 复制 |
| 深度删除 | 经 CD2 删网盘源文件 |
| Emby 302 智能回落 | 115 CDN 失败时可回落 CD2 |

## 检查清单

- [ ] 测试连接成功  
- [ ] 挂载根路径在「文件管理」里能浏览到  
- [ ] 链接同步任务中的「媒体目录 / 云盘根」与 CD2 一致
