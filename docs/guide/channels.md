# 频道与通知

注意区分两件事：

| 概念 | 入口 | 干什么 |
|------|------|--------|
| **频道资源** | 频道页 / 插件「频道」 | 抓取 QQ / Telegram 资源频道 |
| **通知渠道** | 设定 → 通知设置 | 把任务结果推到企业微信等 |

## 频道资源

1. 插件中心启用 **频道**  
2. 进入频道资源页  
3. 用 JSON 导入频道列表，字段大致为 `name` + `id`：  
   - QQ：频道号  
   - Telegram：`@channel` 形式  
4. 按需开关单个频道  

HTTP 出站若需代理，在 **设定 → 基础设置 → HTTP 代理** 统一配置。

## 通知渠道

入口：**设定 → 通知设置**

当前主推：

### 企业微信

| 字段 | 说明 |
|------|------|
| CorpId / AgentId | 应用凭证 |
| 应用秘钥 | Secret |
| Token / EncodingAESKey | 回调相关 |
| 代理 URL | 可选 |
| 管理员白名单 | 可选 |
| 通知事件 | 多选要推送的事件 |

### 微信 ClawBot

| 字段 | 说明 |
|------|------|
| 扫码绑定 | 按页面流程 |
| API 地址 | Bot API |
| 默认接收人 / 白名单 | 按需 |

可另配通知模板（Jinja2）与发送范围（仅操作者 / 管理员 / 全部等）。

## 建议

- 先配好一条通知并手动测发，再打开大量事件  
- 频道资源与通知互不替代：一个是「找片来源」，一个是「结果播报」
