# Emby 配置

入口：**插件中心 → Emby**（媒体服务器 / 302 代理 / 媒体库刷新 / 深度删除）

先配「媒体服务器」，再配 302。

## 1. 媒体服务器

| 字段 | 说明 |
|------|------|
| 名称 | 自定义，如 `家里 Emby` |
| 服务器地址 | 例 `http://192.168.1.100:8096` |
| API Key | Emby → 设置 → API 密钥 |
| 启用 | 开 |

用途：概览媒体库、扫库目标、给 302 选填地址。

## 2. Emby 302 代理（直链播放）

| 字段 | 说明 |
|------|------|
| 名称 | 自定义 |
| Emby 服务地址 | 可从已配媒体服务器选填 |
| 公网地址 | 可选，外网访问时填写 |
| 反代端口 | 可选；**更推荐**直接用 Web 入口下的 `/emby-proxy/{id}`，不必另开端口 |
| 增强功能 | 优化 STRM 首播、外部播放器、美化去广、智能回落（115 CDN → CD2）等按需开 |
| 路径映射 | **本地路径前缀 → 302 目标直链**（核心） |

### 路径映射怎么配

1. 先完成 [链接同步 STRM](./organize.md)  
2. 在 302 页用「从 STRM 配置生成」或「添加映射」  
3. 也可粘贴 `.strm` 内容辅助生成  

### 客户端怎么连

把 Emby 客户端服务器地址改成（端口同 Fast Media Web）：

```text
http://服务器IP:8080/emby-proxy/1
```

`1` 换成列表里的服务器 id / 代理条目。

## 3. 媒体库刷新（可选）

| 字段 | 说明 |
|------|------|
| 启用 | 开 |
| 目标 Emby | 空 = 全部已配服务器 |
| STRM 生成后自动扫库 | 按需 |
| 最少 STRM 数量 | 避免太少就触发 |
| 立即扫库 | 手动点一次验证 |

## 4. 深度删除（可选）

依赖 **设定 → API 令牌**。

1. 填写 Webhook 对外地址  
2. 把生成的 Emby Webhook URL 配到 Emby  
3. 删片时经 CD2 同步删网盘源文件（需 CD2 正常）

## 检查清单

- [ ] 媒体服务器能连上（概览有库）  
- [ ] STRM 已生成且 Emby 能扫到  
- [ ] 302 路径映射已对应挂载前缀  
- [ ] 客户端用代理地址播放成功
