整理与 STRM

入口：整理 & 刮削

Tab用途
全局设置元数据源、电影 / 剧集重命名模板
链接同步生成 STRM，对接 115 + CD2 + Emby
归档刮削监控目录 → 规范命名入库
分类策略YAML 多级目录策略
整理记录归档 / 云下载 / RSS 记录回溯

要 Emby 能播网盘片，通常必须先把 链接同步 跑通。

1. 全局设置

项说明
元数据源TMDB / 豆瓣等
电影重命名格式模板字符串
电视剧重命名格式模板字符串

2. 链接同步（STRM）必填思路

字段说明
任务名称自定义
对应网盘选 115 账号
同步模式常规 / 目录树
网盘目录115 源目录
媒体目录CD2 挂载下的媒体路径
云盘根目录 / 云盘文件夹名如 115open
链接目录生成 .strm 给 Emby 扫描的目录
STRM 模式local 写挂载路径 / cloud 写 CD2 HTTP
后缀、屏蔽词、Cron、启用按需

保存后执行一次同步，确认链接目录出现 .strm。

3. 归档刮削（可选）

规则

名称、目标目录、电影 / 剧集开关。

任务

字段说明
监控目录下载完成落点
目标 / 失败目录归档结果
处理方式移动 / 复制 / cd2 移动 / cd2 复制
归档后生成 STRM建议开
轮询、覆盖、补缺 / 追更按场景

依赖 CD2 时请先保证 CD2 配置 正常。

4. 分类策略

使用 YAML 描述多级分类目录，适合大型库拆分。

与 Emby 的衔接

链接同步输出 STRM
    → Emby 扫「链接目录」
    → Emby 302 路径映射对齐前缀
    → 客户端走 /emby-proxy 播放

详见 Emby 配置。

检查清单

  • [ ] 115 + CD2 已连通
  • [ ] 链接同步产出 .strm
  • [ ] Emby 库能看到对应条目
  • [ ] 302 映射后能播
